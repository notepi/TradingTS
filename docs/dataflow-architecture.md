## 数据流概念层次

数据流分为三个概念层次（不是系统架构层次）：

```
┌─────────────────────────────────────────────────────────────────┐
│ 第一层：Category（主题域）                                        │
│ 用户视角的语义分类                                                │
├─────────────────────────────────────────────────────────────────┤
│  fundamental    │ 基本面数据（财报 + 衍生指标）                   │
│  market         │ 市场数据（股价、成交量）                        │
│  news           │ 新闻数据                                       │
│  indicator      │ 技术指标（MACD, RSI...）                        │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ 第二层：Specification（计算口径）                                │
│ 定义指标是什么                                                    │
│  例：PEG Ratio = PE(TTM) / 净利润增长率                          │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ 第三层：Vendor（数据源）                                          │
│ 不同数据源实现                                                    │
├─────────────────────────────────────────────────────────────────┤
│  tushare  │ A股专用                                              │
│  yfinance │ 全球市场                                             │
└─────────────────────────────────────────────────────────────────┘
```
